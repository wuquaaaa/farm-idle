# 桃源记 · 核心循环设计（小农经济）

> 主题：中式古典田园｜模型：自给自足的小农经济
> 核心命题：**让每一样产物都有"持续被消耗"的去处，而不是一次性解锁后就闲置。**
> 参考：Kittens Game 的"人口—口粮—多链相互制约"，但本土化为小农经济。

---

## 0. 当前已实现

- 资源：🌾粮食 / 🪵木材 / 📄纸 / 📚书籍
- 建筑：农田、林场、造纸坊(木→纸)、书坊(纸→书)、粮仓、小屋
- 知识链：木→纸→书；基础学问耗纸，高深学问耗书籍
- 帮工：农田/伐木分配，吃粮（口粮负反馈雏形）
- 二十四节气：农事/加工产出按季波动
- 转换器引擎：加工建筑按瓶颈比例运转（`consumes` 字段）

下面是把这套地基扩成"完整小农经济"的设计，新增：燃料链、储量链(链D，保留)、铁矿链，并把**人口**抬为核心引擎。

## 一、设计哲学：产物为什么会"一直被消耗"

上一版的教训：`酒 → 钱 → 科技(6个) → 到头`，因为科技是**一次性**的，钱花完就废。小农经济恰恰相反——它的产物大多是**会被用掉、用旧、烧掉、吃掉**的：

- **粮食**被人吃掉（口粮，每天都要）。
- **农具**在耕作中**磨损**，得不断回炉重打。
- **柴薪/木炭**被灶炉**烧掉**。
- **纸**被书写用掉，**书**被读书人翻读、被科举消耗。
- **陶器**作为容器，存粮存水，偶有破损。

所以设计的关键不是"做更多一次性解锁"，而是把产物做成**持续流动的消耗品**与**维护成本**。这样每一条链的下游需求永远开着，上游永远有人要——这正是增量游戏"手停不下来"的来源。

三句话概括目标循环：

> 垦荒种田得粮，粮养人口；人口分工务农、伐木、采矿、读书、做工；
> 木材造纸印书、烧炭为燃料；铁矿配燃料冶铁、打成农具；陶土配燃料烧陶为仓。
> 农具增产却磨损、燃料供烧造、书籍供求学——产物层层相吃，循环不息。

## 二、产物循环消耗总图

```
                 ┌──────────────── 人口（核心引擎）────────────────┐
                 │  住房给名额 · 粮食供养 · 分配为 农夫/樵夫/矿工/工匠/读书人 │
                 └──┬───────────┬──────────┬──────────┬──────────┬──┘
   口粮(每秒吃)◄────┘           │          │          │          │
                    务农        伐木       采矿       做工        读书
                     ▼           ▼          ▼          ▼           ▼
                  🌾粮食       🪵木材      ⛏️铁矿     (操作作坊)   消耗📚书籍
                     │           │          │                      │
          ┌──────────┘     ┌─────┼──────────┘                      ▼
          ▼                ▼     ▼                              📜文气→学问
   养人口/建造/开垦    造纸/烧炭  │
                        │   │    │
                   📄纸 │   │🔥木炭(燃料，烧掉)
                    │   │   │    ├──────────────┐
              📚书籍◄┘   │   │    ▼              ▼
                        │   │  🏺陶窑(黏土+炭)  🔩冶铁炉(铁矿+炭)
                        │   │    ▼              ▼
                        │   │  陶器→仓(扩储量)  🛠️生铁→铁匠铺→农具
                        │   │                    │
                        │   └────────────────────┤(炭被烧掉)
                        ▼                         ▼
                  学问(耗纸/书)            农具装备到 农田/林场
                                          → 增产 ×1.5，但持续【磨损】
                                          → 农具耗尽则掉回 ×1.0
                                          → 逼着一直冶铁补充
```

每个箭头终点都是一个"消耗者"。其中 **口粮、木炭、农具** 三条是**每秒都在消耗**的永久沉淀（见第六节），是整张网张力的来源。

## 四、核心引擎：人口与口粮（负反馈一）

人口是整个经济的发动机，也是 Kittens 的灵魂机制。本土化为"丁口"：

- **名额**由住房决定（小屋→院落→宅院，逐级 +名额，造价递增）。
- **供养**：每口人每秒吃 `0.1` 粮食。养不起就该出走/减产（先用"粮食见底则不再增长"的软约束，后期可做饥荒）。
- **分工**：把人分配到 务农 / 伐木 / 采矿 / 做工(操作作坊) / 读书。每人只干一种活——这制造"该把人往哪放"的核心抉择。
- **张力**：多一口人 = 多一份劳力，但也 = 多一张嘴 + 要更多住房。扩张人口不是无脑好事。

> 现状的"帮工"就是人口雏形（农田/伐木两岗 + 吃粮）。M-next 把岗位扩成 5 类，并让作坊产出与"工匠人数"挂钩。

## 五、各产业链详解

### 链 B · 知识链（已实现）
```
木材 ──造纸坊(+清水)──► 纸 ──书坊──► 书籍
纸 → 基础学问；书籍 → 高深学问
```
循环消耗点：学问消耗纸/书；后期"读书人"持续翻读书籍产出📜文气（让书籍永远有需求，而非研究完即废）。

### 链 C · 燃料链（新）
```
木材 ──炭窑──► 木炭
```
| 建筑 | 投入/秒 | 产出/秒 |
|------|---------|---------|
| 炭窑 kiln(charcoal) | 2 木材 | 1 木炭 |

木炭不直接用，而是**被陶窑与冶铁炉每秒烧掉**——这是连接"木材"与"陶/铁两链"的枢纽，也是一条永久消耗。

### 链 D · 储量链 / 陶冶（保留）
```
黏土 ──陶窑(+木炭)──► 陶器 ──► 建「仓」扩储量
```
| 建筑 | 投入/秒 | 产出/秒 | 说明 |
|------|---------|---------|------|
| 取土场 claypit | — | 0.25 黏土 | 原料 |
| 陶窑 kiln | 2 黏土 + 1 木炭 | 1 陶器 | 烧陶，吃燃料 |
| 仓 warehouse | 造价含陶器 | — | 每座 +各资源储量 |

意图：储量本身成为玩法——产能上来后会**溢出**，逼你回头烧陶建仓。陶器把"黏土+燃料"沉淀成长期储量。

### 链 E · 铁矿链（新 · 循环消耗的核心）
```
铁矿 + 木炭 ──冶铁炉──► 生铁 ──铁匠铺──► 农具
农具 装备于 农田/林场 → 增产，但持续【磨损】
```
| 建筑 | 投入/秒 | 产出/秒 |
|------|---------|---------|
| 矿场 mine(+矿工) | — | 0.2 铁矿 |
| 冶铁炉 furnace | 2 铁矿 + 1 木炭 | 1 生铁 |
| 铁匠铺 smithy | 2 生铁 | 1 农具 |

**农具磨损机制（本设计的点睛之笔）**：
- 每座 农田/林场 每秒消耗约 `0.02` 农具作为"维护"。
- 若农具库存足以支付维护 → 这些建筑产出 **×1.5**（铁器增产）；
- 若农具耗尽 → 维护失败，掉回 **×1.0**。
- 于是：田越多 → 农具维护需求越大 → 冶铁/采矿/烧炭需求越大 → 又需要更多矿工/樵夫 → 又需要更多粮养人……

这条链把"铁"从一次性买卖变成**与农场规模挂钩的永久维护沉淀**，是整张网最强的循环消耗，永不闭合。

## 六、三个永久沉淀（设计的心脏）

整张经济网之所以"手停不下来"，靠三处**每秒都在消耗**的沉淀：

1. **口粮**：人口每秒吃粮。人越多吃越多 → 逼你持续扩田。（约束人口）
2. **农具磨损**：农田/林场每秒磨损农具。田越多磨损越快 → 逼你持续冶铁。（约束铁器增产）
3. **燃料燃烧**：陶窑/冶铁炉每秒烧木炭。炉越多烧得越快 → 逼你持续烧炭、砍木。（约束加工产能）

三处沉淀互相牵引：想多产铁→多烧炭→多砍木→多樵夫→多粮→多田→田又要更多农具…… 任何一处供不上，整条链按瓶颈降速（转换器引擎已支持）。这就是猫国式"瓶颈永远在轮换"的本土化实现。

---

## 七、节气与负反馈（负反馈二）

二十四节气（已实现）叠加在上面，制造全年规划：
- 春催种、夏抢收（芒种农事 +60%）、秋宜酿/宜造（加工 +30~40%）、冬粮减（农事 ×0.5，加工照常）。
- 冬天粮食骤减但口粮照吃 → 逼你"秋囤粮、冬靠存粮+专心加工"。
- 与三处沉淀叠加：冬天田产掉一半，但农具照磨、人照吃 → 是最吃存量的季节。

四道负反馈（口粮 / 工具磨损 / 燃料 / 冬季）共同确保玩家持续做决策，而不是放着自动涨。

## 八、数值与配比（初版，待实测调平）

原料（每座每秒，节气调）：农田 0.5 粮｜林场 0.3 木｜矿场 0.2 矿｜取土场 0.25 黏土｜水井 0.4 水。

加工（投入→产出，每座每秒）：
- 炭窑：2 木 → 1 木炭
- 造纸坊：1 木 + 1 水 → 0.5 纸
- 书坊：2 纸 → 1 书
- 陶窑：2 黏土 + 1 木炭 → 1 陶器
- 冶铁炉：2 铁矿 + 1 木炭 → 1 生铁
- 铁匠铺：2 生铁 → 1 农具

维护/消耗：人口口粮 0.1 粮/人/秒；农具磨损 0.02 农具/(农田或林场)/秒，达标增产 ×1.5。

配比原则：故意让上游略紧（如 1 冶铁炉需 1 木炭 → 需 0.5 炭窑 → 需 1 木/秒），制造持续扩建动机；越下游/越稀缺的建筑造价倍率越高（1.18~1.25）。

## 九、对现有引擎的改动 & 实施路线

引擎基本够用：`consumes` 转换器、节气系数、人口吃粮都已在。新增主要在数据层 + 两个小机制。

| 里程碑 | 内容 | 关键新机制 |
|--------|------|------------|
| **已完成** | 粮/木/纸/书 + 知识链 + 节气 + 帮工 | 转换器、节气系数 |
| **M3a 燃料链** | 木炭(炭窑)；让加工建筑 `consumes` 木炭 | 燃料永久消耗 |
| **M3b 储量链/陶冶** | 取土场/陶窑/陶器/仓（链D） | 陶器→储量；溢出压力 |
| **M3c 铁矿链** | 矿场/冶铁炉/铁匠铺/农具 + **磨损维护** | 工具磨损（核心循环消耗）|
| **M3d 人口扩展** | 岗位扩为 农夫/樵夫/矿工/工匠/读书人；作坊产出挂工匠数 | 分工抉择 |
| **M4 文气深化** | 读书人耗书籍产文气；文气开可叠加升级树 | 让书籍永久有需求 |
| **M5 传家/转生** | 攒够后重开换永久家声 | prestige 终极沉淀 |

建议顺序：**M3a 燃料链 → M3c 铁矿链(含农具磨损) → M3b 储量链 → M3d 人口**。先把"工具磨损"这条循环消耗做出来，验证它是否真能让铁/木/矿三链一直转。
