# 桃源记 · 新循环设计文档

> 主题：中式古典田园｜核心机制：多层加工资源链（参考 Kittens Game）＋ 二十四节气、火候品质两套原创特色
> 目标：在不重构现有引擎的前提下，把"两资源单线"扩成"多资源加工网"，制造猫国式的"瓶颈轮换 + 解锁好奇心"。

---

## 一、设计目标

现状的循环是 **粮食 →（卖）→ 金币 → 买建筑/科技 → 更多粮食**，本质是一根线、一个瓶颈，缺乏长期张力。

新循环要解决三件事：

1. **从"单线"变"加工网"**：低级原料必须被加工成中级、高级材料，高级建筑/成品反过来吃这些材料，形成相互制约的经济网。
2. **永远卡在某个瓶颈上**：让玩家不断在"现在缺什么"之间做权衡，而不是放着自动涨。
3. **本土化的节奏与深度**：用二十四节气替代猫国的四季，用"火候/品质"加一条垂直于"数量"的成长轴，做出辨识度。

设计原则：**数据驱动**。新内容尽量只加在 `data/` 配置表里，少改 `systems/` 逻辑——这正是现有架构的强项。

---

## 二、新核心循环总览

```
        ┌──────────────────── 二十四节气（全局调味）────────────────────┐
        │  春催种 · 夏抢收 · 秋宜酿 · 冬囤货，按节气给各类产出加/减成      │
        ▼                                                              ▼
  采集原料 ──加工──► 初级材料 ──再加工──► 成品 ──卖/进献──► 铜钱 / 文气
   稻谷            精米              米酒                      │
   木材            柴薪              绸缎          ┌───────────┘
   清水            生丝              糕点          ▼
   桑叶            陶器(储量)        茶            买更高级加工建筑 + 解锁科技
   黏土                                            │
        ▲                                          ▼
        └──────── 火候/品质：成品有 粗/中/精/极品 四档，决定售价与文气 ◄──┘
```

一句话：**采原料 → 逐层加工 → 卖成品赚铜钱 / 进献换文气 → 扩产 + 解锁下一层加工链**，节气调节产出节奏，火候决定每件成品的"含金量"。

## 三、资源体系（分层）

资源分四层。"层"是设计概念，引擎里都是同一种 `ResourceState`。

| 层级 | 资源 | 图标 | 来源 | 主要用途 |
|------|------|------|------|----------|
| T0 原料 | 稻谷 grain | 🌾 | 农田 / 点击 | 口粮、碾米原料 |
| T0 原料 | 木材 wood | 🪵 | 林场 / 伐木帮工 | 建造、烧柴、陶窑 |
| T0 原料 | 清水 water | 💧 | 水井 / 沟渠 | 灌溉、酿造、加工 |
| T0 原料 | 桑叶 mulberry | 🍃 | 桑园 | 养蚕（生丝链）|
| T0 原料 | 黏土 clay | 🟤 | 取土场 | 陶器（储量链）|
| T1 初级材料 | 精米 rice | 🍚 | 碾房（稻谷→精米）| 酿酒、做糕、口粮升级 |
| T1 初级材料 | 柴薪 firewood | 🔥 | 柴房（木材→柴薪）| 酿造/焙茶燃料 |
| T1 初级材料 | 生丝 silk | 🧵 | 蚕室（桑叶→生丝）| 织绸 |
| T1 初级材料 | 陶器 pottery | 🏺 | 陶窑（黏土+柴薪→陶器）| **提升各资源储量上限** |
| T2 成品 | 米酒 wine | 🍶 | 酒坊 | 高价售卖、进献文气 |
| T2 成品 | 绸缎 cloth | 🧧 | 织坊 | 高价售卖、进献文气 |
| T2 成品 | 糕点 pastry | 🥮 | 糕点铺 | 中价售卖、节庆热销 |
| T2 成品 | 香茶 tea | 🍵 | 焙茶坊 | 中价售卖、进献文气 |
| 货币 | 铜钱 gold | 💰 | 卖成品 | 买建筑/帮工、升火候 |
| 抽象 | 文气 culture | 📜 | 私塾 / 进献成品 | **解锁科技树** |

设计要点：
- **陶器**是储量调节器（对应猫国的仓库/谷仓逻辑），它本身要加工，于是"扩容"也成了玩法。
- **文气**是科技货币（对应猫国的 science/文化）。它有两条来源：私塾被动产出，或把高级成品"进献"换取大量文气——给玩家一个"卖钱 vs 换科技"的取舍。

## 四、加工链详解（核心）

加工建筑是**转换器**：每秒消耗输入、产出输出。若输入不足则按比例降速运转（瓶颈自动显现）。配比是设计灵魂，下面给出初版数值（每座建筑每秒，dt 已计入）。

### 链 A：主食链（最早开放）
```
农田 ─► 稻谷 ──碾房─► 精米 ──酒坊（+清水+柴薪）─► 米酒
                  └──糕点铺（+糖/蜜）─► 糕点
```
| 加工建筑 | 投入 / 秒 | 产出 / 秒 | 备注 |
|----------|-----------|-----------|------|
| 碾房 mill | 2 稻谷 | 1 精米 | 主食链入口 |
| 酒坊 winery | 3 精米 + 2 清水 + 1 柴薪 | 1 米酒 | 高利润成品 |
| 糕点铺 bakery | 2 精米 + 1 清水 | 1 糕点 | 节庆溢价 |

### 链 B：燃料链（支撑酿造/焙茶）
```
林场 ─► 木材 ──柴房─► 柴薪
```
| 加工建筑 | 投入 / 秒 | 产出 / 秒 |
|----------|-----------|-----------|
| 柴房 woodshed | 2 木材 | 1 柴薪 |

### 链 C：丝绸链（中期）
```
桑园 ─► 桑叶 ──蚕室─► 生丝 ──织坊─► 绸缎
```
| 加工建筑 | 投入 / 秒 | 产出 / 秒 |
|----------|-----------|-----------|
| 蚕室 silkhouse | 4 桑叶 | 1 生丝 |
| 织坊 weavery | 3 生丝 | 1 绸缎 |

### 链 D：储量链（中期，调节上限）
```
取土场 ─► 黏土 ──陶窑（+柴薪）─► 陶器 ─► 仓房（提升储量）
```
| 加工建筑 | 投入 / 秒 | 产出 / 秒 | 备注 |
|----------|-----------|-----------|------|
| 陶窑 kiln | 2 黏土 + 1 柴薪 | 1 陶器 | 陶器存入仓房，每个 +储量 |

### 链 E：茶链（中后期）
```
茶园 ─► 茶叶 ──焙茶坊（+柴薪）─► 香茶
```
| 加工建筑 | 投入 / 秒 | 产出 / 秒 |
|----------|-----------|-----------|
| 焙茶坊 tea-roaster | 3 茶叶 + 1 柴薪 | 1 香茶 |

**瓶颈轮换效果**：开了酒坊就缺精米 → 多建碾房 → 碾房吃稻谷又缺稻谷 → 多开农田/调更多帮工务农 → 又缺柴薪烧酒……每加一层，前面所有资源的需求被重新激活，玩家手永远停不下来。这正是猫国耐玩的根。

## 六、科技树（文气解锁）

科技用**文气**解锁，是控制内容节奏的总闸门。建议解锁链（→ 表前置）：

```
碾磨之术 ─► 酿造 ─► 蚕桑 ─► 织造
   │          │
   └► 水利     └► 陶冶 ─► 茶事
   └► 营造法式（降造价）
   └► 节气历法（强化节气增益）
   └► 火候（解锁品质系统）
   └► 传家（解锁转生，后期）
```

| 科技 | 文气花费 | 前置 | 效果 |
|------|----------|------|------|
| 碾磨之术 | 30 | — | 解锁 碾房 / 柴房 |
| 水利 | 50 | 碾磨 | 农田产出 ×2、解锁水井升级 |
| 酿造 | 80 | 碾磨 | 解锁 酒坊 / 糕点铺 |
| 蚕桑 | 120 | 酿造 | 解锁 桑园 / 蚕室 |
| 织造 | 180 | 蚕桑 | 解锁 织坊 |
| 陶冶 | 150 | 酿造 | 解锁 取土场 / 陶窑 / 仓房 |
| 茶事 | 200 | 陶冶 | 解锁 茶园 / 焙茶坊 |
| 营造法式 | 160 | — | 所有建筑造价 −15% |
| 节气历法 | 100 | — | 节气增益幅度 +50% |
| 火候 | 140 | 酿造 | 解锁品质系统（见第八节）|
| 传家 | 400 | 织造+茶事 | 解锁转生（见第九节）|

> 现有 `TECHS` 的 `effects` 已支持 `multiply_production / unlock_building / storage_boost`，新增需要一个 `unlock_resource_chain` 或直接用多个 `unlock_building` 效果即可，改动很小。

## 七、原创特色一：二十四节气历法

**替代猫国的四季**，更细腻、更中式。

- **历法推进**：游戏内设"日"，每 `N` 秒 = 1 天（建议 1 天 = 真实 6 秒）；15 天 = 1 个节气；24 节气 = 1 年（≈ 36 分钟一轮）。
- **每个节气施加一组产出系数**，按季分组：

| 季 | 节气 | 典型效果 |
|----|------|----------|
| 春 | 立春·雨水·惊蛰·春分·清明·谷雨 | 稻谷/桑叶等**种植产出 +20%~40%**；谷雨当节大幅 +50% |
| 夏 | 立夏·小满·芒种·夏至·小暑·大暑 | 芒种**抢收**：稻谷 +60% 但清水消耗 +30%；大暑加工降速 |
| 秋 | 立秋·处暑·白露·秋分·寒露·霜降 | 秋分前后**酿造/加工 +30%**（宜酿）；霜降原料产出 −20% |
| 冬 | 立冬·小雪·大雪·冬至·小寒·大寒 | 稻谷等**原料产出大减（−40%~−60%）**，但加工链不受影响 → 鼓励秋囤原料、冬天专心加工 |

**策略意图**：制造"春扩田、夏抢收、秋开酿、冬囤货加工"的全年规划压力，让放置之外多一层主动决策。科技"节气历法"放大增益、削弱减益。

**实现**：新增 `CalendarSystem`（实现 `GameSystem`），`tick` 里累加 `dayTimer`、推进 `state.calendar.{day, termIndex, year}`，并把当前节气的系数表暴露给 `BuildingSystem` 在算产出时乘进去。UI 在 `ResourceBar` 显示当前节气 + 一句农谚。

## 八、原创特色二：火候 / 品质

**垂直于"数量"的第二条成长轴**，是和猫国最大的区分点。

- 每种**成品**（米酒/绸缎/糕点/香茶）有品质档：**粗(×1) · 中(×1.5) · 精(×2.5) · 极品(×4)**，品质决定**售价与进献文气量的倍率**。
- 品质来源（取最简实现）：每座加工建筑有一个"火候等级"，花**铜钱**升级（铜钱回收池），火候越高，该建筑产出的成品品质档越高。
- 进阶可选：品质受**节气**影响（秋分出的酒更醇）、受专门投入影响（多耗一份柴薪/清水提火候）。

**策略意图**：给后期玩家一个"把铜钱再投资"的深坑——是铺更多酒坊（走量），还是把现有酒坊火候拉满（走质）？两条路都通向更高收益，但节奏不同。

**实现**：`BuildingState` 增加 `qualityLevel` 字段；新增轻量 `QualitySystem` 或直接在 `BuildingSystem` 产出时按 `qualityLevel` 查倍率表。售卖/进献时读取成品对应的品质倍率。

## 九、原创特色三（可选 · 后期）：传家转生

对应猫国 Paragon，留作后期留存核心，本期可只留接口。

- 满足条件（如累计进献文气 ≥ 阈值）后可"传家"：重置本局产业，按本局成就换取永久资源**家声**。
- 每点家声给 **+1% 全局产出**，并解锁"祖业"小科技树（永久降价、解锁开局赠送等），跨局保留。
- 主题包装：分家立业，把一身本事传给下一代——重开不是惩罚，是越传越旺。

## 十、数值与节奏曲线（初版手感）

- **开局 0~5 分钟**：点击/农田攒稻谷 → 建碾房出精米 → 私塾出文气 → 解锁酿造。即时正反馈。
- **5~20 分钟**：主食链跑顺（农田→碾房→酒坊），卖酒赚第一桶铜钱，铺燃料链。第一个明显瓶颈：精米/柴薪供不上酒坊。
- **20~60 分钟**：开蚕桑/陶冶两条新链，文气科技加速，节气规划开始有感。
- **1 小时+**：火候系统拉深度，多链并行管理，逼近第一次传家。
- 所有加工配比遵循"**上游产能需 ≥ 下游需求**"，故意让默认配比略紧（如 1 酒坊需 3 碾房供米），制造持续扩建动机。
- 成本倍率统一 1.15~1.22：链越靠下游、越稀缺的建筑倍率越高，避免无脑刷。

## 十一、对现有引擎的改动清单

好消息：地基齐全，多数改动集中在**数据层**。

**0）先修一个已存在的不一致（建议第一步）**
`state.ts` 的类型定义落后于实现：`workerSystem.ts` 已经用了 `state.workers.{foodPerSec, allocatedFarmland, allocatedLumber}`、`state.buildings.hut`、`state.resources.wood`，但 `state.ts` 的 `Workers/Buildings/Resources` 接口里没有这些 → 当前严格类型下编译不过。**先把 `state.ts` 补齐对齐**，再往上加新内容。

**1）`core/state.ts`**：在 `Resources/Buildings/Techs` 接口与 `createInitialState` 里补齐新资源（water/mulberry/clay/精米/柴薪/生丝/陶器/米酒/绸缎/糕点/香茶/文气）、新建筑、新科技；`BuildingState` 加 `qualityLevel`；新增 `calendar` 字段。

**2）`data/` 配置表**：
- `resources.ts`：补全所有新资源定义。
- `buildings.ts`：`BuildingDef` **新增 `consumes?: Partial<Record<资源, number>>` 字段**（加工建筑的每秒投入），把上面所有建筑录入。
- `techs.ts`：录入第六节科技；`TechEffect` 可加 `unlock_building` 复用即可。

**3）`systems/buildingSystem.ts`（唯一核心逻辑改动）**：`tick` 里给有 `consumes` 的建筑加**转换器逻辑**——
```
需求 = count × dt
对每个输入 res：可供比 = min(1, 现有量 / 需要量)
有效比 = 所有输入可供比的最小值      ← 瓶颈
按 有效比 扣除输入、产出输出 × 节气系数 × 火候倍率
```
无 `consumes` 的建筑走原有"无中生有"产出逻辑，向后兼容。

**4）新增 `systems/calendarSystem.ts`**：推进节气、输出当前产出系数。注册进 `engine` 即可（架构本就支持即插即用）。

**5）`systems/marketSystem.ts`**：从只卖稻谷扩成可卖多种成品，并支持"进献成品换文气"；售价读品质倍率。

**6）`components/`**：`ResourceBar` 显示新资源 + 节气；`BuildView` 展示加工建筑的投入/产出与火候升级按钮；可新增 `CalendarView` / 把进献并入 `MarketView`。

**7）存档版本**：`SaveManager` 已有 `version` 字段与导入/导出；新增字段时把 `version` 升到 2，并在 `load` 里对旧档做字段补默认值（迁移函数），避免老玩家存档崩。
